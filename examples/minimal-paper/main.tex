\section{Why This Exists}

This minimal paper demonstrates a tight loop for tuning Python-generated figures in the context that matters: the final PDF layout.

\section{Figure In Context}

The figure below is intentionally included at column width. In the current prototype, the figure script itself is the editable surface, so typography and visual choices can be tuned directly in Python and reflected in the paper immediately.

% fontbigger: demo
\begin{figure}[t]
  \centering
  \includegraphics[width=\columnwidth]{figures/output.pdf}
  \caption{A matplotlib figure regenerated from the active figure script.}
  \label{fig:demo}
\end{figure}

Small parameter tweaks are usually obvious only after the figure is embedded in the paper. This prototype shortens that feedback loop.

\section{Multiple Figures}

The sample paper also includes a second figure generated from a converted existing script. This demonstrates the multi-figure workflow: each figure can have its own script, output path, and registry entry.

% fontbigger: prefetcher
\begin{figure}[t]
  \centering
  \includegraphics[width=\columnwidth]{figures/prefetcher/learner_sensitivity_bar_chart.pdf}
  \caption{A converted prefetcher sensitivity chart managed through the figure registry.}
  \label{fig:prefetcher}
\end{figure}

With the cursor inside either figure environment, the command `Font Bigger: Open Figure Script In VS Code` opens the matching script and switches the active figure.
